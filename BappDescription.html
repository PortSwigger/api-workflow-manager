API Workflow Manager centralizes API endpoint management and organization during security testing. It enables security professionals to collect, categorize, and export HTTP requests with workflow labels, supporting structured testing methodologies and comprehensive documentation through CSV and cURL exports.

Features

  • Collect and organize APIs from Proxy/Repeater
  • Add workflow markers to structure testing phases
  • Filter and manage endpoints by HTTP method
  • Export to manual testing sheets or cURL commands
  • Control duplicate entries for clean inventories

Usage

  1. Right-click any request in Proxy, Repeater, or HTTP history and select Extensions → Send APIs to Management Tab to collect endpoints
  2. Use the API Management tab to view collected APIs, apply HTTP method filters, and organize entries with Move Up/Down buttons
  3. Add workflow markers using the Add Workflow button to create named testing phase boundaries within your API list
  4. Configure duplicate handling using the Allow duplicate APIs checkbox to maintain clean inventories or preserve all requests
  5. Export filtered APIs using the Export Filtered APIs button, choosing between CSV format for manual testing sheets or cURL format for command-line testing
  6. Use context menu options to directly export selected requests as CSV manual sheets or cURL commands without adding them to the management tab
