API & Workflow Manager is a Burp Suite extension that aims to centralize management and organization for API endpoints during security testing. Through this extension, security professionals can collect, categorize, and export HTTP requests with workflow labels, facilitating structured testing methodologies and comprehensive documentation through CSV and cURL exports.

Features

Core Management

  • Centralized API Management → Collect and organize HTTP requests in a dedicated tab.
  • Smart Filtering → Filters APIs based on HTTP methods such as GET, POST, PUT, DELETE, etc.
  • Duplicate Control → Configurable duplicate detection is featured to maintain clean API inventories.
  • Visual Workflow Creation → Add workflow markers to structure testing processes.

Quick Actions

  • Context Menu Integration → Direct access from Proxy/Repeater.
  • Drag & Drop Reordering → Intuitive UI for organizing API sequences.
  • Bulk Operations → Multi-select support for efficient management.

Export Capabilities

  • CSV Manual Sheets → Generates documentation for manual testing.
  • cURL Commands → Converts requests to cURL format for external testing.
  • Flexible Formats → Multiple export options with duplicate handling.

Usage

API Management Tab

The API Management Tab allows you to manage your captured APIs efficiently with the following operations:

  • View all collected APIs.
  • Filter by HTTP method.
  • Reorder or remove entries.
  • Add new APIs or workflows manually.

Context Menu Integration

Right-click any request in Proxy, Repeater, or HTTP history, and choose either:

  • Send APIs to Management Tab
  • Export API's as Manual Sheet
  • Export cURL to File

CSV Export Example

The exported CSV file includes structured columns for:

  • API & parameters
  • Attack steps & observations
  • Manual test tracking (Found/Not found)

cUrl Export Example

Requested API endpoints can also be exported in cURL format for different testing scenarios outside Burp Suite.

Duplicate Filtering to avoid noise

The extension is built with a feature that can filter out duplicate entries, reducing noise during the API testing and management process.

Usage Guide

Adding APIs to Management

From Context Menu:

Right-click on request → Extensions → Send APIs to Management Tab

Manual Addition:

Use the Add API button in the management tab. Enter Method, URL, and Parameters manually.

Filtering:

You can filter displayed APIs by applying filters based on the HTTP methods.
